If the PoC uses a standalone Rust program that calls Move VM or compiler APIs directly (e.g., `MoveVM::new()`, `Session::execute_function()`, `Compiler::new()`, `move_compile()`) instead of invoking the `aptos move` subcommands, it is not an acceptable PoC. The bug must be triggered through an `aptos move` subcommand (`compile`, `test`, `run-script`, `disassemble`, `decompile`, etc.) using crafted Move source or bytecode as input. Additionally, the triggering code path must originate in `third_party/move/` — bugs whose stack traces live entirely outside that subtree are out of scope.
