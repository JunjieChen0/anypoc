If the PoC uses standalone C/C++ programs that call Redis library functions or client library functions (e.g., hiredis `redisConnect()`, `redisCommand()`) instead of using the built-in Redis binaries (`redis-server`, `redis-cli`), it is not an acceptable PoC. If the PoC requires loading a custom Redis module (via `MODULE LOAD`), it is not an acceptable PoC. The bug must be triggered through the standard Redis binaries and built-in commands without custom modules.
